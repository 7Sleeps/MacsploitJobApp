Getting Started

Prerequisites

Ensure you have a compatible system (details provided upon purchase) and have acquired execution credits from the Pricing page.

Installation & Setup

Download the Hydrotax client application using the link provided after purchasing credits. Follow the platform-specific installation instructions. You will need to configure the client with your API key (found in your account dashboard) for authentication.

# Example CLI configuration (replace with actual command)
 hydrotax config --api-key YOUR_API_KEY_HERE

Your First Execution

Prepare your script (e.g., `my_script.lua`). Use the Hydrotax client or API to submit it for execution. Each submission consumes one execution credit.

# Example CLI execution
 hydrotax execute --script my_script.lua --port 7777

The command will return an execution ID. You can use this ID to retrieve logs and results via the client or API.

# Example CLI log retrieval
 hydrotax logs --id EXECUTION_ID_HERE

API Reference (Conceptual)

Hydrotax provides a RESTful API and potentially language-specific SDKs for programmatic interaction.

Authentication

All API requests must include your API key, typically via an `Authorization: Bearer YOUR_API_KEY` header.

Endpoints

  • POST /v1/executions: Submit a script for execution. Requires script content, configuration parameters (like port). Returns an `executionId`.
  • GET /v1/executions/{id}: Get the status of an execution (pending, running, completed, failed).
  • GET /v1/executions/{id}/logs: Retrieve the logs for a completed or running execution.
  • GET /v1/executions/{id}/results: Retrieve any defined results or artifacts from a completed execution.
  • GET /v1/credits: Check your remaining execution credit balance.

Scripting Environment API (Inside Execution)

Scripts running within Hydrotax have access to a specific API (e.g., Lua modules) for interacting with the environment:

-- Example Lua script using Hydrotax API
 local htx = require('hydrotax')

 htx.log.info("Script started.")

 -- Example: Interact with simulated network on configured port
 local connection = htx.network.connect(htx.config.port)
 if connection then
   connection:send("DATA_PACKET")
   local response = connection:receive()
   htx.log.debug("Received: " .. response)
   connection:close()
 end

 -- Example: Define a result artifact
 htx.results.set("output_data", { success = true, value = 42 })

 htx.log.info("Script finished.")

Note: This is a conceptual overview. Detailed API specifications and SDKs would be provided with the actual service.
